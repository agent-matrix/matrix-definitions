# Ruslan Magana Definitions

Ruslan Magana Definitions, abbreviated **RMD**, are the original control rules for Matrix Builder, agent-generator, MatrixHub, and the RuslanMV AI-coding ecosystem.

Industry standards explain common best practices. RMD explains the unique philosophy of this ecosystem:

> AI coders are workers, not architects.

## Core RMD principles

1. Every generated project must have a blueprint contract.
2. Every generated project must have a standards lock.
3. AI coders may only work inside task-scoped boundaries.
4. Architecture drift is rejected.
5. New dependencies require approval.
6. Control files are immutable after generation.
7. Repair prompts are generated after failed validation.
8. MatrixHub publishes only validated artifacts.

## RMD naming

RMD rules use IDs like:

```text
RMD-001
RMD-102
RMD-301
```

Suggested ranges:

| Range | Meaning |
|---|---|
| RMD-001 to RMD-099 | Core blueprint and standards-lock rules |
| RMD-100 to RMD-199 | AI-coder control rules |
| RMD-200 to RMD-299 | Agent safety rules |
| RMD-300 to RMD-399 | MatrixHub publication rules |
| RMD-900 to RMD-999 | Branding and attribution rules |
